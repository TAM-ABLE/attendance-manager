# メール配信設定についてのご相談

## 背景

勤怠管理システムでは、ユーザー招待メールやパスワードリセットメールを送信する機能があります。
現在、本番環境でメール送信に失敗しており、ユーザー登録ができない状態です。

対応として、メール配信サービス **Resend**（https://resend.com）を使う方式に切り替えます。
コードの書き換えは自分で対応しますが、以下の対応にご協力をお願いいたします。

---

## お願いしたいこと

### 1. Resendアカウントについて

メール配信サービス Resend（https://resend.com）のアカウントが必要です。
自分の会社メールアドレスで作成して進めても問題ないでしょうか？それとも管理者の方で作成すべきでしょうか？

- 登録に必要なもの：メールアドレスのみ
- 無料プラン（月3,000通）で問題ありません

### 2. DNSレコードの追加

Resendから `tamable.co.jp` ドメインでメールを送るには、DNSレコードの追加が必要です。
Resendの管理画面でドメインを登録すると、追加すべきレコードが表示されます。

- [ ] `tamable.co.jp` のDNSを管理しているサービスはどれですか？（例：お名前.com、Cloudflare、AWS Route 53 など）
- [ ] DNS管理画面にアクセスできる方はどなたですか？

### 3. APIキーの共有

Resend管理画面からAPIキーを発行して共有してください。
本番環境の環境変数に設定して使用します。

### 4. 送信元アドレスの決定

メールの差出人アドレスをどれにするか決めてください。

- 例：`noreply@tamable.co.jp`、`info@tamable.co.jp` など
- [ ] どのアドレスを送信元にすべきですか？

---

ご確認よろしくお願いいたします。
